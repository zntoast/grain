# Grain 综合创作预设设计

## 目标

用一批覆盖人物、商业视觉和世界构建的综合创作预设，替换当前内置的“动漫创作、写实摄影、概念设计”及其基础词组。新版预设应开箱即用，并且不会删除或覆盖用户自行创建的数据。

## 数据范围

新版内置数据包含：

- 9 个工作空间，放入 3 个一级目录。
- 24 个词组，放入 5 个一级目录。
- 每个词组预关联 6 至 16 个现有提示词。
- 每个工作空间预关联 6 至 10 个词组，并标记为正向或负向。

目录保持单层即可满足本次预设；现有二级目录能力不变。

词组优先复用 `DEFAULT_TAGS` 和 `NOVELAI_TAGS` 中已经存在的短词。关联时使用稳定的提示词 ID，不复制提示词实体。负面排除只关联适用于多数模型的通用负面词；若现有词库不足，可补充少量短词，但不引入长句提示词。

## 内置数据标识

所有新版内置实体采用稳定且带命名空间的 ID：

- 工作空间：`preset_ws_*`
- 工作空间目录：`preset_wsf_*`
- 词组：`preset_group_*`
- 词组目录：`preset_gf_*`

旧版内置工作空间 ID 为 `ws_main`、`ws_photo`、`ws_concept`；旧版内置词组 ID 为 `character`、`scene`、`style`、`lighting`、`composition`、`quality`、`pose`、`clothing`、`expression`、`effect`。这些 ID 构成明确的替换清单，不能通过名称猜测用户数据。

## 升级与持久化

快照版本从 `1` 提升到 `2`，并在加载浏览器数据或本地 JSON 数据时执行同一套迁移：

1. 若快照版本低于 `2`，移除替换清单中的旧版内置工作空间、词组及其关联。
2. 删除只属于旧预设的目录映射；用户目录和用户实体不变。
3. 合并新版内置工作空间、词组和目录，ID 已存在时保留当前实体，缺失时补入默认值。
4. 为新版内置实体补齐默认关联；不改写用户实体的关联。
5. 若当前工作空间指向被移除的旧预设，切换到新版第一个工作空间；否则保持当前选择。
6. 将版本写为 `2`，后续加载不再重复迁移。

用户主动删除某个新版预设后，不应在每次刷新时复活。因此加载版本为 `2` 的快照时只做结构规范化，不再无条件合并预设。未来若需要增加新一批内置内容，应通过新的版本迁移完成。

## 代码边界

- `constants.ts`：定义新版预设实体、目录、映射和关联。
- `types.ts`：将快照版本扩展为 `1 | 2`，便于读取旧数据并输出新版数据。
- `store.ts`：集中实现纯函数式的 v1 到 v2 迁移，并让浏览器存储、本地文件导入共用。
- 新增迁移测试：覆盖旧预设替换、用户数据保留、重复加载幂等、当前工作空间回退和已升级快照不复活已删除预设。

## 错误处理

- 缺失字段继续使用现有规范化逻辑补默认值。
- 关联到不存在实体的旧内置记录在迁移时清理。
- 用户数据中的未知字段保持兼容，不因迁移失败清空整个快照。
- 本地文件读取失败沿用现有提示和浏览器数据回退流程。

## 验收标准

- 新安装首次打开可看到 3 个工作空间目录、9 个工作空间、5 个词组目录和 24 个词组。
- 每个工作空间有可直接组合提示词的默认词组。
- 从旧版升级后，旧内置预设不再出现，用户创建的数据完整保留。
- 同一份数据重复加载不会重复创建目录、工作空间、词组或关联。
- 用户删除新版内置项后刷新页面，该项不会自动恢复。
- TypeScript 构建和相关测试通过。
