# Grain UI 视觉美化设计

## 目标

在不改变现有信息架构、功能逻辑、路由和数据结构的前提下，将 Grain 提示词管理界面升级为轻盈、专业、带柔和粉色品牌感的桌面工具界面。

## 视觉方向

- 风格：轻盈专业型，克制使用粉色，避免大面积甜腻渐变。
- 色彩：中性灰白作为主体，玫粉作为主操作和选中状态，紫色仅用于少量分类辅助。
- 层级：以背景、边框、轻阴影和留白建立层次，减少依赖高饱和色块。
- 形状：统一 10–14px 圆角；胶囊标签保持紧凑，按钮与输入框高度统一。
- 字体：保留系统中文字体栈，强化标题、正文、辅助信息之间的字号和字重差异。

## 交互参考与转译

参考 Animal Island UI 的圆润控件与明确状态反馈，但不复制其森系配色、插画背景和重拟物装饰。

- 按钮：借鉴全圆角、轻微底部投影和按下回弹感。Grain 主按钮使用 10–12px 圆角而非完全胶囊形，避免工具界面显得过于玩具化；小型图标按钮可使用圆形或 10px 圆角。
- 按压反馈：默认态保留 1–2px 的柔和底部阴影，按下时向下位移 1px 并收起阴影；悬停只提升颜色和边框，不做夸张漂浮。
- 图标：延续现有 Lucide 图标库，建立 14 / 16 / 18 / 20px 四档尺寸；同类操作保持相同线宽、容器尺寸和文字间距。
- 图标控制：纯图标操作统一使用 32–36px 点击热区，并提供 `title` 或可访问名称；主要按钮内图标固定放在文字前或后，不混用方向。
- 选择与筛选：借鉴圆润边框、清晰选中态和禁用态。选中项采用品牌浅粉底加深粉文字，避免大面积实色填充。
- 动效：采用 160–220ms 的缓动；图标只在有语义时进行轻微旋转、缩放或位移，避免常驻弹跳。

## 改造范围

### 全局基础

- 统一页面背景、文本色、边框色、阴影、圆角、焦点环和过渡动画。
- 清理脚手架遗留样式，建立可复用的视觉工具类。
- 保证键盘焦点清晰，并兼容 `prefers-reduced-motion`。

### 侧边栏与布局

- 侧边栏采用柔和暖白背景和细分隔线，降低粉色面积。
- 强化当前页面选中态，统一导航项、目录项和数量徽标。
- 优化拖拽分隔条的可发现性，同时保持现有宽度调整逻辑。

### 首页

- 收紧首屏内容宽度，增强品牌标题和主按钮层级。
- 三张统计卡使用一致的图标容器、边框和悬停反馈。
- 保留当前内容与操作，不新增营销模块。

### 工作空间页

- 统一页头、统计卡、筛选控件、汇总区和词组卡片的间距与层级。
- 减少卡片内部碎片化色块；正向、负向状态仍有明确语义色。
- 强化主要操作按钮，弱化次要操作和危险操作。

### 词组详情与提示词管理

- 优化上传预览区、提示词文本区、已选标签区和分类筛选区的边界。
- 标签采用紧凑胶囊形态，英文词与中文释义保持可扫读性。
- 大量标签页面通过更清楚的分类标题、行距与吸顶工具栏提升浏览效率。

### 弹窗与通用组件

- 统一弹窗遮罩、容器阴影、标题区、关闭按钮、输入框和操作区。
- 统一 Button、Modal、SearchBox、TagChip、GroupCard 等通用组件的状态表现。
- 为按钮、图标按钮、选择控件建立统一的 hover / active / focus-visible / disabled 状态。

## 交互约束

- 不改变任何现有点击、拖拽、搜索、增删改、复制、路由和本地存储行为。
- 不引入新的运行时依赖，不替换现有图标库。
- 不改变桌面端信息密度；小屏仅修复明显溢出，不进行移动端重构。
- 动效限于 120–220ms 的颜色、阴影和轻位移过渡。

## 验收标准

- 首页、工作空间、词组详情、全部提示词和弹窗形成统一视觉语言。
- 主要操作、当前选中项、普通信息和危险操作的视觉优先级清晰。
- 现有功能与数据行为保持不变。
- `npm run build` 与 `npm run lint` 通过。
- 在浏览器中逐页检查常见桌面视口，确认无明显遮挡、溢出或错位。
