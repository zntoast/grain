## Context

Grain 当前使用 Zustand 管理工作空间、词组、提示词及关联关系，并将完整快照写入 localStorage 或用户选择的 JSON 文件。工作空间只能按词组开关并复制汇总文本，相关临时状态未持久化；默认提示词在加载时会自动补齐。

## Goals / Non-Goals

**Goals:**

- 保持本地优先，不引入服务端和新运行时依赖。
- 为旧快照提供无感默认值，继续支持导入导出和文件同步。
- 将重复检测、排序、权重格式化和历史裁剪放在纯函数中测试。
- 编排状态按工作空间隔离，不修改词组和提示词原始内容。
- 默认提示词合并或删除后不在下次加载时复活。

**Non-Goals:**

- 不实现别名、同义词、自定义标签、使用次数和最近使用统计。
- 不新增独立拼装页面。
- 不提供在线分享、云同步或多人协作。
- 不为不同生图平台提供多套权重语法。

## Decisions

1. **工作空间编排使用独立配置**
   - 使用 `workspacePromptConfigs[workspaceId]` 保存禁用词组、提示词顺序、禁用提示词和权重。
   - 原因：同一个词组可被多个工作空间复用，不能把临时编排写回词组。
   - 替代方案：扩展 `workspaceGroups` 每个关联项。未采用，因为单个提示词配置会让关联结构过重。

2. **提示词使用稳定派生键**
   - 标准提示词键为 `<groupId>:tag:<tagId>`，自定义行键为 `<groupId>:custom:<index>`。
   - 原因：同一个提示词在不同词组中可以独立调整。
   - 权衡：修改自定义行顺序可能改变其键，旧配置会自然失效。

3. **预设和历史保存完整快照**
   - 保存当前 `workspaceGroups` 和 `WorkspacePromptConfig`。
   - 原因：恢复时必须同时还原词组状态、顺序和单词配置。
   - 历史在复制时写入，按最新优先全局裁剪到 30 条。

4. **重复检测使用标准化英文**
   - 比较时去除大小写差异、空格、下划线和连字符。
   - 用户明确选择保留项，其他 ID 在所有词组关联中替换为保留 ID 并去重。

5. **默认提示词删除使用墓碑记录**
   - `deletedTagIds` 记录被删除或合并的默认提示词 ID。
   - 加载时仍可补充新增默认数据，但不得补回墓碑中的 ID。

## Risks / Trade-offs

- [大量提示词使编排弹窗较长] → 使用分区、固定高度和内部滚动；后续可增加搜索。
- [跨分类同英文词可能语义不同] → 合并必须由用户逐组确认，不自动执行。
- [自定义行索引变化导致配置失效] → 失效配置不会影响数据，重新编排即可。
- [删除词组后预设含失效引用] → 派生提示词时忽略不存在的词组；活动配置同步清理。

## Migration Plan

1. 旧快照缺少新增字段时初始化为空数组或空对象。
2. 首次保存后写入新字段，不改变快照版本和旧字段语义。
3. 回滚代码时旧版本会忽略未知字段，原工作空间、词组和提示词仍可读取。
